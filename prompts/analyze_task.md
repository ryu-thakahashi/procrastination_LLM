## タスク分析の実施

与えられたタスクの情報をもとに、タスク分析を実施してください。

## 制約

- `preventing_factors` の各値（task_expectations / task_value / feedback_time）は **20字以内** で記述すること。

## タスク情報

{{
    "title": "{task_name}",
    "description": "{task_desc}",
    "selected_cause": "{selected_cause}",
}}

## 例1

### 入力

{{
    "title": "論文執筆",
    "description": "Social Networks に出す論文の執筆を先延ばしにしてしまう。",
    "selected_cause": "感情的なハードル: 完成した時に受けるフィードバックが怖い",
}}

### 出力

{{
    "title": "論文執筆",
    "description": "Social Networks に出す論文の執筆を先延ばしにしてしまう。",
    "selected_cause": "感情的なハードル: 完成した時に受けるフィードバックが怖い",
    "minimum_goal": "Social Networks に出す論文の初稿を保存する",
    "preventing_factors": {{
        "完了させる気がしない": "ポジティブなフィードバックを貰える気がしない",
        "できなかった時の損失が大きい": "ネガティブフィードバックを受けたときの損失を過大推定",
        "フィードバックまでの時間が長い": "ポジティブフィードバックを貰えるクオリティになるまで、時間がかかりすぎる",
    }}
}}

## 例2

### 入力

{{
    "title": "就活のES作成",
    "description": "サイバーエージェントに提出するESの作成を先延ばししてしまう。",
    "selected_cause": "感情的なハードル: 完成させたのに不合格になったときのことを考えると怖い",
}}

### 出力

{{
    "title": "就活のES作成",
    "description": "サイバーエージェントに提出するESの作成を先延ばししてしまう。",
    "selected_cause": "タスク粒度が粗い:やらなきゃいけないタスクはわかっているが、そのタスクを完了させるまでが長い",
    "minimum_goal": "サイバーエージェントのESを1社分、項目を埋めた状態で保存する",
    "preventing_factors": {{
        "完了させる気がしない": "タスク（e.g., 自己PRの完成）が長すぎて、完了させられる気がしない",
        "できなかった時の損失が大きい": "タスク（e.g., 自己PR）が合否を分ける、最も重要な項目であると感じる",
        "フィードバックまでの時間が長い": "タスク（e.g., 自己PR）の完成まで時間がかかり、達成感を得にくい",
    }}
}}
