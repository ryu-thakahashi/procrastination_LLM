# チャクシュ — 先延ばし防止レポート生成 AI

> **先延ばしにしているタスクを入力するだけで、あなたの背中を押すレポートを AI が生成します。**

---

## 1. アプリのコンセプト

| | |
|---|---|
| **アプリ名** | チャクシュ（着手） |
| **一言紹介** | "やらなきゃ" を "やってみよう" に変える、先延ばし防止レポート生成 AI |

アプリ名の **「着手」** は、先延ばし防止の唯一の解が「最初の一歩を踏み出すこと」であるという思想を表しています。チャクシュは完璧なタスク管理を目指すのではなく、**着手する勇気を生む**ことだけにフォーカスします。

---

## 2. 背景・課題

「やらなきゃいけないのに、どうしても手が動かない」——そんな経験は誰にでもあります。

先延ばしの研究（Temporal Motivation Theory など）によれば、人がタスクを先延ばしにする原因は大きく **3 つの認知バイアス** に分類できます。

| 要因 | 内容 | 例 |
|---|---|---|
| **達成期待の低さ** | 「自分にはどうせ無理」という思い込み | 初稿を完成させられる気がしない |
| **損失の過大推定** | 失敗したときの損失を必要以上に大きく見積もる | 完璧でないと落ちる気がする |
| **フィードバックの遠さ** | 達成感を得るまでの時間が長すぎる | 結果が出るのが遠くてやる気が出ない |

既存ツールはタスクの「整理」は得意ですが、これらの **心理的ブロックを解消する**機能は持っていません。

| ツール名 | 特徴 | 課題 |
|---|---|---|
| **Todoist / Notion** | タスクをリスト・データベースで整理できる | 「やること」は見えても「なぜできないか」には答えてくれない |
| **Pomodoro タイマー** | 25 分集中・5 分休憩のサイクルで作業を習慣化する | そもそも着手できない人には効果が薄い |
| **ChatGPT / Gemini** | 自由な対話でアドバイスをもらえる | 自分で適切な質問を考える必要があり、先延ばし要因の網羅的な診断は難しい |

---

## 3. ソリューション

チャクシュは、先延ばしの心理的要因を **AI が自動診断し、個別の対策を提示**します。

```
タスク名・説明を入力
        ↓
① タスク分析     ── 最小ゴール・マイルストーン・先延ばし要因を特定
        ↓
② 対策立案       ── 各要因への具体的なカウンタープランを生成
        ↓
③ レポート生成   ── 共感ベースの文章で背中を押すレポートを出力
```

レポートの最後には **「あなたがやるべき、たった一つのこと」** として、そのまま生成 AI にコピペできるプロンプトを提示。読み終えた瞬間に行動に移せる設計になっています。

---

## 4. 主要機能

### 先延ばし要因の自動診断
タスク名と説明を入力すると、Gemini が以下を自動で抽出します。
- **最小ゴール**: 完璧でなくてもよい、今回達成すべき最低ラインの定義
- **マイルストーン**: ゴールまでの工程を小さなステップに分解
- **先延ばし要因**: 達成期待・損失認知・フィードバック時間の 3 軸で診断

### パーソナライズされたレポート
3 ステップのパイプライン（分析 → 対策 → レポート生成）で、入力したタスクに完全特化した文章を生成。汎用的なアドバイスではなく、**あなたの悩みに直接刺さる言葉**で届けます。

### リアルタイム進捗表示
レポート生成中は Server-Sent Events (SSE) で進捗をリアルタイム配信。「分析中」「対策立案中」「生成中」の 3 ステップをアニメーションで可視化します。

### コピペ可能なプロンプト
レポート末尾にはそのまま生成 AI に投げられるプロンプトを掲載。コピーボタン一発でクリップボードに保存できます。

## 5. 技術スタック

| レイヤー | 技術 |
|---|---|
| **バックエンド** | Python 3.12 / FastAPI / Uvicorn |
| **AI モデル** | Google Gemini 2.5 Flash (`google-genai`) |
| **フロントエンド** | HTML / Tailwind CSS (CDN) / marked.js / Vanilla JS |
| **ストリーミング** | Server-Sent Events (SSE) |
| **環境管理** | uv / python-dotenv / Pydantic |
| **デプロイ** | Render |

### アーキテクチャ概要

```
browser
  │  POST /api/report  (task_name, task_desc)
  │◄─── SSE stream ────────────────────────┐
  │                                         │
FastAPI (main.py)                           │
  │                                         │
  ├── analyze_task()      → 01_task_analysis.md
  ├── generate_task_strategy() → 02_task_strategy.md
  └── generate_report()   → 03_report.md ──┘
         │
    Gemini 2.5 Flash API
```

AI への指示はすべて `prompts/` 以下の Markdown ファイルで管理しています。プロンプトを Python コードから分離することで、ロジックを変えずに AI の振る舞いだけを調整できます。

## 6. こだわったポイント

### 心理学ベースのプロンプト設計
「先延ばし」を感情論で語るのではなく、**Temporal Motivation Theory** の構成要素（達成期待・タスク価値・フィードバック遅延）に基づいて要因を構造化。プロンプトでは JSON スキーマを明示し、AI の出力を安定させています。

### AI と人間の役割分担の明示
対策の手順において「**AI**：AIができること」「**あなた**：あなたにしかできないこと」を明示的に区別。ユーザーが迷わず動き出せるよう、行動の主体を明確にしています。

### 読みやすいレポート表示
- **目次（TOC）** を自動生成してレポート全体を俯瞰しやすく
- **マイルストーンをタイムライン形式**で可視化（`<ol>` を独自スタイルに変換）
- **コードブロックにコピーボタン**を自動付与

### SSE によるリアルタイム体験
Gemini の応答待ちを「待ち時間」ではなく「生成の過程」として演出。`asyncio.to_thread` でブロッキングな Gemini API 呼び出しを非同期化し、各ステップが完了するたびに UI を更新します。

### モックモードによる開発効率化
`USE_MOCK=true` を設定すると過去の出力を返すモードに切り替わり、API コストをかけずに UI の開発・確認ができます。

## 7. 今後の展望

- **ユーザー認証・履歴保存**: 過去のレポートをふりかえり、先延ばしのパターンを可視化する
- **タスク進捗トラッキング**: マイルストーンの完了をチェックし、達成感をリアルタイムで提供する
- **マルチターン対話**: チャット形式で先延ばし要因を深掘りし、より精度の高い対策を生成する
- **プッシュ通知**: 設定した時間に「今日はこのマイルストーンを完了しよう」とリマインドする
